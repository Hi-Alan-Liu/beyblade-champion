# 陀螺配置卡片產生器 — 團隊協作日誌

> 規則：每個角色開工前先讀完本檔案，做完把成果寫回這裡。
> 輪巡順序：**PM → UIUX → 工程師 → QA → PM(擴張建議)** → 不停輪巡直到喊停。
> 每輪約 1 小時（每角色約 15 分鐘）。

## 專案總綱（Round 0 / PM 立案）

**產品**：純前端的「陀螺配置卡片」快速產生器，輸出可下載的 PNG 圖片。
**靈感來源**：4 張比賽名次卡（1ST~4TH PLACE，HANIK TRACK）。

### 卡片內容拆解（從參考圖逆向）
- 左上角：賽事 LOGO 文字（HANIK TRACK），顏色隨名次變化
- 左側：人物照片（可上傳）
- 左下角：超大名次字（1ST / PLACE 兩行），白色描邊 + 名次主色
- 右側：深色圓角面板（含淡淡賽道底圖，可更換），內含 3 顆陀螺
- 每顆陀螺一橫排，由左到右 3 個零件圖：上蓋(組裝體) / 固鎖(齒盤) / 軸心(軸尖)
- 每排下方：合併名稱文字，例：`隕星龍騎士8-70L`
- 右下角：地區/賽事浮水印（例：高雄漢諡）

### 名次主色
| 名次 | 顏色 |
|------|------|
| 1ST | 紅 #E10600 |
| 2ND | 藍 #1455C0 |
| 3RD | 綠 #1A9E3E |
| 4TH | 紫 #7A3FD4 |

### 零件命名規則
合併名稱 = `上蓋名` + `固鎖名` + `軸心名`，例：上蓋「隕星龍騎士」+ 固鎖「8-70」+ 軸心「L」= `隕星龍騎士8-70L`

### 硬性限制
- 不需要資料庫
- 純前端，可離線
- 重點是「快速生成 + 下載圖片」

## Round 1

### 🎩 PM — Round 1（立案與範圍）

**MVP 範圍（這輪一定要做出來）**
1. 名次選擇（1ST~4TH），自動套用主色到 LOGO / 名次字 / 強調色
2. 文字輸入：LOGO 文字、名次副標(暱稱)、右下浮水印
3. 人物照上傳（左側）
4. 背景底圖上傳（右側面板，可更換；不傳就用預設深色）
5. 3 顆陀螺，每顆 3 個零件圖上傳 + 上蓋/固鎖/軸心名稱輸入
6. 即時預覽（所見即所得）
7. 一鍵下載 PNG（高解析度）

**使用流程（User Flow）**
```
進站 → 選名次 → 填文字 → 上傳人物照 → (選填)換背景
     → 逐顆陀螺：上傳3零件圖 + 填3零件名
     → 預覽確認 → 下載 PNG
```

**優先級（MoSCoW）**
- Must：名次配色、零件圖/名輸入、預覽、PNG 下載
- Should：人物照與背景上傳、合併名稱自動組合、解析度 2x
- Could：範本切換、拖曳排序、本機暫存（localStorage）
- Won't（本輪不做）：登入、雲端儲存、後端、多語系

**驗收標準（給 QA 的檢查點）**
- 切換名次 → LOGO 與名次字顏色即時改變
- 上傳任一圖 → 對應位置即時顯示
- 填零件名 → 合併名稱即時更新
- 點下載 → 取得一張包含所有元素的 PNG，文字不破版

**技術建議（給工程師參考，非強制）**
- 版面用 HTML/CSS 排（好維護），輸出用 `html-to-image` 或 `html2canvas` 擷取 DOM 成 PNG
- 純靜態三檔：`index.html` / `styles.css` / `app.js`
- 卡片固定比例 4:5（1080×1350 輸出，編輯時 540×675 顯示，下載 2x）

**交棒給 UIUX**：請定義左右分欄版面、面板圓角/間距、字體層級、名次配色的視覺套用方式，以及零件三圖的排列與連接線樣式。

### 🎨 UIUX — Round 1（版面與視覺規格）

**整體畫面（雙欄）**
```
┌─────────────────────────────────────────────┐
│ 左：控制面板 (可捲動表單)  │  右：卡片預覽 (置中、sticky) │
└─────────────────────────────────────────────┘
```
手機 (<900px) 改為上下堆疊：預覽在上、表單在下。

**卡片內部版面（1080×1350，4:5）**
- 底：深色 `#0d0d10`
- 左上 LOGO：x=40 y=36，字體 800 斜體，字級 ~30px，色=名次主色，字距加寬
- 左側人物照：佔卡片左 ~46% 寬，底部對齊，`object-fit: contain` 不裁切人臉
- 左下名次字：兩行（如 `1ST` / `PLACE`），字級超大 (~120px / ~96px)，900 斜體，主色填充 + 4px 白描邊，輕微陰影
- 右側面板：絕對定位於右 ~52% 區，圓角 28px，半透明深色 `rgba(20,20,26,.92)`，內含背景底圖 (cover, 透明度 ~0.35)
  - 內距 28px，3 列等高，列間連接圓點 + 細虛線（左緣）
  - 每列：grid 3 欄 → `[上蓋 1.2fr] [固鎖 1fr] [軸心 .8fr]`，圖片 contain、置中
  - 列底名稱：靠右、白字 700、字級 ~30px，前面一個小三角刻度（▸ 樣式）
- 右下浮水印：紅底白字斜貼紙感（或主色），字級 ~30px

**字體層級**
| 元素 | 字重 | 字級(1x) | 說明 |
|------|------|----------|------|
| 名次大字 | 900 斜 | 60/48px | 視覺主角 |
| LOGO | 800 斜 | 15px | 品牌 |
| 零件名稱 | 700 | 15px | 可讀優先 |
| 浮水印 | 700 | 15px | 角落 |
| 表單標籤 | 600 | 14px | 控制面板 |

字型：標題用 `'Arial Black', 'Noto Sans TC', sans-serif`（斜體運動感）；中文用 `Noto Sans TC`。

**配色套用（CSS 變數 `--accent`）**
- LOGO 文字色、名次字填充、浮水印底、面板列連接點、表單焦點框 → 全部綁 `--accent`
- 切換名次 = 換 `--accent` 一個變數即可全卡同步

**空狀態 / 預設**
- 未上傳圖 → 顯示淺色虛線框 + 「上蓋 / 固鎖 / 軸心」提示文字（不會出現在下載圖中？→ 決策：保留淡灰提示，讓空卡仍可讀；QA 注意）
- 未上傳人物 → 左側留空（保持版面）

**互動細節**
- 所有 input 即時綁定預覽（input 事件）
- 圖片上傳用 `FileReader` 轉 dataURL（純前端、可離線、可被擷取）
- 下載按鈕固定在表單頂部，明顯主色

**交棒給工程師**：請以三檔 (`index.html`/`styles.css`/`app.js`) 實作上述版面；卡片以 1080×1350 的 `.card` 容器用 `transform: scale()` 縮放顯示；輸出用 html-to-image 的 `toPng` 並設 `pixelRatio` 取得高解析度；圖片一律走 dataURL 避免 CORS 污染 canvas。

**交棒給 QA**：請用 browse 開啟 `index.html`，驗證：名次切換配色、文字即時更新、合併名稱、預覽版面是否破版；並截圖佐證。圖片上傳因 headless 無法手動選檔，請用程式注入測試圖驗證版位。
